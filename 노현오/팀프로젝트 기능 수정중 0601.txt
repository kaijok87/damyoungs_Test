= 영상 재생 또는 이미지 를 보여주고  끝나면 인게임으로 복귀하는 기능

- 설청 창 
	- 새로 시작(게임내 초기화가 필요한 데이터초기화 작업)
		- 초기화에 필요한 데이터의 리스트업이 필요
	-  이어하기 
		- 저장방식이 정해지면 데이터를 불러와서 인풋작업(게임내 스크립트로 임시 저장공간(Class)를 만들어서 데이터를 넣고 잘들어가는지 체크 )
	-  저장(인게임 데이터를 외부데이터로 출력작업) 
		- 데이터를 외부에 저장시 저장하는 방식을 지정 (파일로 출력할것인지 데이터베이스를 만들것인지  :  전자가 작업량이 적음  , 파일로저장할때 데이터의 저장구조를 어떻게만들지 XML추천) 		
	=  화면 설정(게임내 공통된 내용(소리,화면배율설정,키설정,프레임속도등)들 조절할수있게 인터페이스 제작 및 기능구현 )
	- 종료(프로그램 종료)  


- 캐릭터 생성 창
	- 캐릭터 종류 고를수있게 인터페이스 작업및 데이터 인풋작업과 이름 생성작업
	2023/05/23 1교시내용추가 
		- 커스터마이징 

- 인게임 (기본적으로 화면구성 과 기능및 데이터 으로 나뉜다.)
	- 화면에 상시 보이는 것들  =[옵션에서 온오프 가능하게 조절 또는 이동도중에는 오프하다가 전투나 액션이있을때  온]
		- 캐릭터 상태 뷰(체력 ,배고픔 ,목마름 등등) 
		= 나침판 및 지도 , 시계  +
			= 현재 지도 및 전체 지도 변환 가능한 기능
	- 캐릭터의 행동 
		- 이동		달리기 구현
		- 전투 		공격모션을 만들고 공격오브젝트로 충돌시 타격임팩트 이거나 아니면 공격 범위 체크하여 타격임팩트 제작
		= 콤보		공격모션끝나고 연달아서 스킬을 사용하거나 다시공격을 할시 다른모션을 보여주면서 타격처리도 다르게한다
		= 스킬 ,마법 	모션과 임팩트를 만들고 전투와 같은 방식으로 만든다
		- 채집 	채집모션과 전투와 비슷한 매커니즘으로 처리 필드오브젝트와의 상호작용
				

	- 캐릭터 능력치 및 상태창
		- 능력치 
			-  힘민지체
				- 능력치에 맞는 스킬 구성이 필요
			- 상태이상 [상태이상의 가감도 등을 추가로 설정하여 회복시간이나 풀리기위한 자원이 추가로 들수있도록 처리]
 				-  독  		시간변화에따른 데미지 처리기능 구현 , 아이템 또는 자동회복같은 회복 처리
				- 마비  	특정시간 동안 캐릭터만 시간처리를 안함  
				- 빙결   	마비와 동일 애니메이션 처리만 틀림
				- 수면  	마비와 같은 처리기능을 가지되 캐릭터에게 액션이 들어오면 회복 처리
				- 화상  	독과 같은 특징을 지니되  특정 액션을 하면  회복 되는 처리 추가 
				- 동상	 	화상과 같은 기능 애니메이션만 틀림.		
				- 감기  	스테미나 다는속도가 늘어난다던가 캐릭터 시간처리 속도를 조금늦춘다던가
				- 상처     	특정 시간또는 회복할때까지 시간처리동안 생명력을 깍던가 전투나 자원수집속도를 늘린다.
				- 슬로우    	캐릭터의 시간처리를 낮춘다 . 특정시간동안 지속되거나 회복이되면 정상속도로돌아간다.
			- 버프
				- 능력치 증가 특정 시간동안 특정 능력치를 증가시키는 기능
				- 상태방지     특정 상태 발동시 상태이상을 무시하게 하는기능
				- 헤이스트   캐릭터빼고 시간처리를 느리게 만들어서 상대적으로빠르게 하는기능 또는 캐릭터의 시간처리속도를 늘려서 처리하는기능
				- 속성부여 (화상 , 빙결 , 독 ,추가데미지 등) 공격시 위의 상태이상을 발동시킬수있게 하는기능 
	-  인벤토리 창 
			전체 장비 도구 아이템 음식 책(스크롤) 재료 퀘스트 
			- 창을 탭형식으로 분할하거나 하나의 창으로 만들고 정렬기능을 추가 및 스크롤기능 추가
			- 아이템을 클릭해서 특정 위치로 이동하여 인덱스 저장 (인벤토리에 보여줄 아이템순서를저장할 배열을 저장 )
			- 인벤토리 크기조절아이템변경시 데이터 관리 (배열크기조절또는 배열크기를 인벤토리아이템(야생소년의 가방류)별로 만들어서교체)
			
	
	- 제작 또는 강화 창
			- 아이템 스토리지클래스 에 습득한 제작리스트를 관리할 변수를 추가하던가 아니면 다른 클래스로 새로 제작
			  
	- 아이템 
			- 아이템 스토리지클래스(싱글톤으로제작)를 작성하여  유저가 가지고있는 데이터만 관리
			- 
				 
	-  도감 .
			= 아이템에대한 정보를 읽어오고 습득정보를 유저캐릭터 가 자기고있도록 추가한다.
